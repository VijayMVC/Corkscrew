Corkscrew.SDK.objects Namespace
This namespace contains the object hierarchy of the Corkscrew CMS system
Classes
  Class Description
CSBaseCollectionT
Base collection class to implement collections of Corkscrew objects. All methods except constructors and Dispose are thread-safe. NOTE: Implementation is borrowed from implementation of .NET Framework "Collection<T>" class.
CSBaseCollectionTCSBaseCollectionEnumerator
Enumerator implementation for CSBaseCollection
CSConfigurationCollection
Collection of configuration key-pairs. Elements are CSKeyValuePair.
CSDiagnosticsEntry
This class represents a single entry in the diagnostics/tracing backend table of JournalLog. Allows the caller to write new messages to the diagnostics backend.
CSFarm
CSFarm is the top-most level item in the primary Corkscrew object hierarchy. All other constructs are treated as members or sub-constructs of the CSFarm object. A farm may contain a collection of files and directories, but these are actually transparently hosted under the "Configuration" or "ConfigDB" CSSite.
CSFileSystemEntry
This is a base class. Represents a single file or directory item in the Corkscrew filesystem. Most of the useful functionality is only accessible from within an implementing object (CSFileSystemEntryFile and CSFileSystemEntryDirectory).
CSFileSystemEntryCollection
Collection of CSFileSystemEntry objects
CSFileSystemEntryDirectory
Represents a directory type entry in the Corkscrew filesystem, derives from CSFileSystemEntry class.
CSFileSystemEntryDirectoryCollection
Collection of CSFileSystemEntryDirectory
CSFileSystemEntryFile
Represents a file type entry in the Corkscrew filesystem, derives from CSFileSystemEntry class.
CSFileSystemEntryFileCollection
Collection of CSFileSystemEntryFile
CSFileSystemHistoryRecord
Represents a single historical record of a CSFileSystemEntry. The data in this class cannot be modified, hence it does not contain any methods.
CSKeyValuePair
Our own implementation of a KeyValuePair, since the .NET KeyValuePair is a struct and a Tuple is too verbose. This is used from our CSConfigurationCollection class.
CSMailItem
Represents a single email object. This is primarily used by the automated system to create and send emails. Can be used by workflows and other components that require email capabilities.
CSMIMEType
Defines a map between a file extension and a MIME/content Type
CSMIMETypeCollection
Collection of CSMIMEType
CSPath
This is not a persisted class. Provides functionality similar to System.IO.Path class, but for our CMS file system.
CSSite
A CSSite (Corkscrew Site) is a collection element under the CSFarm object (that is, the CSFarm is the parent of the CSSite). The CSSite hosts the CMS filesystem and provides the collection of files and directories.
CSSiteCollection
Collection of CSSites. Allows management of sites in the farm.
CSSiteDNSNamesCollection
Collection of DNS names for a CSSite
CSSiteHistoryRecord
Represents a single historical record of a CSSite. The data in this class cannot be modified, hence it does not contain any methods.
CSTable
Defines a table structure in Corkscrew. Tables are stores for row/column data just like a database table. In fact, in the background, the class is persisted as an actual database table.
CSTableCollection
Collection of tables
CSTableColumn
Represents a single column that has been added to a table
CSTableColumnCollection
Represents all the columns added to a particular table
CSTableColumnCollectionChangedEventArgs
Event arguments for the ColumnAdded and ColumnRemoved events
CSTableColumnDefinition
Defines a column for a Corkscrew Table. This is only the definition and will not contain any data. Column definitions are created at Site scope. These must then be added to one or more tables to make use of them.
CSTableColumnDefinitionCollection
Collection of table column definitions
CSTableRow
Defines a row in a Corkscrew Table (CSTable). Tables are stores for row/column data just like a database table. The table row class represents one row of data in the table.
CSTableRowCollection
Represents all the rows added to a particular table
CSTabularDataColumnDataTypeConverter
Helps converting between the column data type Enum and actual database type name
CSWorkingContext
Provides context information to different layers of Corkscrew.
PATH_INFO
Used to return or pass in information about paths

IWorkflowAssociable
Interface. Classes that implement this interface can be associated with a workflow
IWorkflowInstantiable
Interface. When classes that implement this interface are created, modified or deleted; they cause workflow instances to be spawned.
